    50→
    51→1. /api/teachers/route.ts - GET list (with search + status filters), POST create (User + Teacher transactional, bcrypt password hash, auto-generate TCH-2024-XXX ID)
    52→2. /api/teachers/[id]/route.ts - GET (with marks/assignments/attendance), PUT (owner or teacher.edit perm, syncs user table), DELETE (super_admin/admin only, removes user + teacher)

    54→4. /api/marks/[id]/route.ts - GET, PUT (mark.edit; recomputes grade when score/total changes), DELETE (mark.delete)
    55→5. /api/attendance/route.ts - GET list (filters: studentId, teacherId, date, status; date filter uses gte/lt for full-day match), POST create (attendance.create)
    56→6. /api/attendance/[id]/route.ts - PUT (attendance.edit), DELETE (attendance.delete)
    57→7. /api/assignments/route.ts - GET list (filters: teacherId, subjectId, status; includes subject/teacher/submissions), POST create (assignment.create; auto-resolves teacherId from current user's teacher profile if not provided)
    58→8. /api/assignments/[id]/route.ts - GET (with submissions), PUT (assignment.edit), DELETE (assignment.delete)
    59→9. /api/submissions/route.ts - GET list (filters: assignmentId, studentId; students limited to own submissions), POST create (assignment.submit; students auto-bound to their studentId)
    60→10. /api/submissions/[id]/route.ts - PUT (assignment.edit; used for grading - score/feedback/status)
    61→11. /api/ranks/route.ts - GET (returns saved ranks if they exist, otherwise computes live from marks sorted by totalScore; supports term/academicYear/gradeId filters; Rank model has no student relation so student+grade data fetched separately and merged), POST (rank.create; upserts per student/term/year)
    62→12. /api/reports/route.ts - GET (report card: student + marks for term + computed totals/average + saved rank + live class rank by comparing to classmates + attendance summary; students restricted to own reports)
    63→13. /api/promotions/route.ts - GET list (filters: studentId, academicYear; includes student), POST create (promotion.create; optionally updates student's gradeId when promoted=true)
    64→14. /api/grades/route.ts - GET list with sections, subjects count, and active/total student counts, ordered by level
    65→15. /api/subjects/route.ts - GET list (filter: gradeId; includes grade info)
    66→
